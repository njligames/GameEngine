387 /*!***************************************************************************
388  @class PVRShell
389  @brief Inherited by the application; responsible for abstracting the OS and API.
390  @details PVRShell is the main Shell class that an application uses. An
391  application should supply a class which inherits PVRShell and supplies
392  implementations of the virtual functions of PVRShell (InitApplication(),
393  QuitApplication(), InitView(), ReleaseView(), RenderScene()). Default stub
394  functions are supplied; this means that an application is not
395  required to supply a particular function if it does not need to do anything
396  in it.
397  The other, non-virtual, functions of PVRShell are utility functions that the
398  application may call.
399  *****************************************************************************/

424  /*!***********************************************************************
425  @brief Initialise the application.
426  @details This function can be overloaded by the application. It
427  will be called by PVRShell once, only at the beginning of
428  the PVRShell WinMain()/main() function. This function
429  enables the user to perform any initialisation before the
430  render API is initialised. From this function the user can
431  call PVRShellSet() to change default values, e.g.
432  requesting a particular resolution or device setting.

443  @return true for success, false to exit the application

445  virtual bool QuitApplication() { return true; };

447  /*!***********************************************************************
448  @brief Initialise the view.
449  @details This function can be overloaded by the application. It
450  will be called by PVRShell after the OS and rendering API
451  are initialised, before entering the RenderScene() loop.
452  It is called any time the rendering API is initialised,
453  i.e. once at the beginning, and possibly again if the
454  resolution changes, or a power management even occurs, or
455  if the app requests a re-initialisation.
456  The application should check here the configuration of
457  the rendering API; it is possible that requests made in
458  InitApplication() were not successful.
459  Since everything is initialised when this function is
460  called, you can load textures and perform rendering API functions.
461  @return true for success, false to exit the application
462  *************************************************************************/
463  virtual bool InitView() { return true; };
464 
465  /*!***********************************************************************
466  @brief Release the view.
467  @details This function can be overloaded by the application. It
468  will be called after the RenderScene() loop, before
469  shutting down the render API. It enables the application
470  to release any memory/resources acquired in InitView().
471  @return true for success, false to exit the application
472  *************************************************************************/
473  virtual bool ReleaseView() { return true; };
